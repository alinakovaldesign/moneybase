# DESIGN-001 — Claude Design: explore the four key moments

**Goal**: 2–3 distinct visual directions for the highest-judgment moments of the flow, explored on canvas BEFORE any token is locked in code.
**Surface(s)**: Claude Design canvas (platform-neutral at this stage; nativeness comes later via tokens).
**Inputs**: assessment brief, **DESIGN-000 research output** (`docs/research.md`: Moneybase brand cue sheet + B2B pattern adopt/avoid list), brand adjectives (regulated, calm, precise, B2B operator — not consumer-playful), `02-design-system/tokens/tokens.seed.json` as a starting palette only.
**Output**: canvas exports + prompt log in `evidence/design-sessions/DESIGN-001/`, committed; verdict table below filled in.

## The four moments
1. Wallet home — wallet card anatomy, per-currency balance presentation
2. Wizard shell — stepper, density, how "regulated calm" reads in a form
3. Card-link explainer + consent — the trust-critical screen; how honesty looks visually
4. Success moment — celebration appropriate for B2B money movement (restrained, informative)

## Acceptance
- [ ] ≥2 genuinely distinct directions per moment (not palette swaps) — divergent layout/hierarchy/tone
- [ ] Every canvas iteration driven by a written prompt; prompts logged verbatim in `evidence/design-sessions/DESIGN-001/prompts.md` (this is the Claude Design half of the agent-driving evidence)
- [ ] Exports (PNG per direction + canvas share link) committed under this directive
- [ ] Verdict per direction: keep / kill / merge, with one line of reasoning each
- [ ] At least one direction killed with a stated reason (exploration without rejection is not exploration)
